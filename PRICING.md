# Pricing and Credits

OmniForge AI uses license keys with monthly credit allocations. Credits are OmniForge usage units, not raw provider tokens. The backend handles model routing, provider costs, rate limits, and safety checks behind the scenes.

## How Credits Work

- Simple chat, story, and design turns usually cost fewer credits.
- Agent turns that edit files, call tools, compile, inspect assets, or use stronger model routes cost more.
- Vision, search, large context, and long tool loops may cost additional credits.
- Credits reset on the billing cycle for active subscriptions.
- Rate limits protect service availability; higher tiers receive larger burst and weekly limits.

## Model Selection

Most users should leave model selection on `auto`. OmniForge chooses the best available route for the current task based on the license tier, mode, and backend policy.

Manual model selection is available on higher tiers when enabled for the license. The plugin only shows public OmniForge model names; provider details remain managed by the OmniForge backend.

## Fair Use

OmniForge is built for real Unreal Engine development work. Abuse, resale of license keys, automated scraping, or attempts to bypass the OmniForge backend can lead to license suspension.
